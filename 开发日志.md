- 新增认证相关文件：
  - `backend/app/api/auth.py`

  - `backend/app/core/deps.py`

  - 路径：`storage/workspaces/{user_id}/default/`

- 工具执行根目录从全局 `WORKSPACE_ROOT` 改为当前 run 绑定的 workspace `root_path`，确保不同用户 workspace 隔离。

- 新增 Workspace 相关文件：
  - `backend/app/api/workspaces.py`
  - `backend/app/services/workspace_service.py`
  - `backend/app/schemas/workspace.py`
- 新增接口：

- `WorkspaceService` 提供核心方法：

  - `get_default_workspace(user_id)`
  - `get_owned_workspace(user_id, workspace_id)`
  - `ensure_workspace_path(workspace)`
- 自定义工作区目录规则：
  - `storage/workspaces/{user_id}/{workspace_id}/`

  - `storage/workspaces/{user_id}/default/`
- `UserService` 只保留用户查询和创建逻辑，workspace 逻辑迁移到 `WorkspaceService`。
- `/api/auth/me` 改为通过 `WorkspaceService` 查询默认 workspace。

- 工具执行仍然只使用当前 run 绑定 workspace 的 `root_path`。
- 当前代码里还没有独立 `DockerRuntime` 实现；后续新增 DockerRuntime 时，需要同样只挂载当前 workspace 的 `root_path`。

### v1 Chat Gateway

- 新增 Gateway 层文件：
  - `backend/app/gateway/gateway_service.py`
  - `backend/app/gateway/request_context.py`
  - `backend/app/gateway/session_router.py`
  - `backend/app/gateway/run_router.py`
- 新增 `backend/app/schemas/chat.py`，把 `ChatRequest`、`ChatResponse`、`RunResponse` 从 API 层移到 schemas。
- `backend/app/api/chat.py` 变薄：`POST /api/chat` 只负责 FastAPI 依赖注入，然后调用 `GatewayService.handle_chat()`。
- `GatewayService.handle_chat()` 负责完整入口编排：
  - 构建 `RequestContext`
  - 解析 workspace
  - 写 `chat.request.received` / `gateway.auth.checked` / `workspace.resolved`
  - 解析 session
  - 检查 active run
  - 保存 user message
  - 创建 run
  - 回填 user message 的 `run_id`
  - 写 `run.created`
  - 投递 RabbitMQ
  - 写 `run.queued`
  - 返回 `session_id` / `run_id` / `status`
- `SessionRouter` 负责创建新 session 或校验已有 session 是否属于当前 user + workspace。
- `RunRouter` 负责 active run 检查和 queued run 创建。
- 由于 `event_logs.workspace_id` 有外键约束，Gateway 会先解析 workspace，再写 request/auth/workspace 相关事件；事件含义不变，但保证落库合法。

### Gateway Router 教学注释

- 给 `backend/app/gateway/session_router.py` 补充教学注释，说明：
  - 为什么 session 解析必须发生在 workspace 解析之后。
  - 新 session 和旧 session 复用的判断路径。
  - 为什么必须校验 session 同时属于当前 user 和当前 workspace。
- 给 `backend/app/gateway/run_router.py` 补充教学注释，说明：
  - Session 和 Run 的职责区别。
  - 为什么同一个 session 先限制只能有一个 active run。
  - 为什么 Run 必须绑定 user、workspace、session。
