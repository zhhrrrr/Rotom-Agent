### RabbitMQ / Worker 升级

- RabbitMQ 消息保持 v0 设计，只投递：
  - `{"run_id": "run_xxx"}`
- 明确 RabbitMQ 只负责通知，PostgreSQL 才是事实来源，不把 `user_id`、`workspace_id`、`message` 塞进消息体。
- Worker 消费 run_id 后新增安全判断：
  - 消息体非法：记录日志并 ack。
  - run 不存在：ack。
  - run 状态不是 `queued`：ack，防止 RabbitMQ 重复投递导致重复执行。
  - run 是 `queued`：写 `worker.run.received`。
  - 将 run 状态改为 `running`，并写 `run.started`。
  - 调用 `AgentOrchestrator.run(run_id)`。
  - Worker 自身异常时将 run 标记为 `failed`，写 `run.failed`，然后 ack。
- `AgentOrchestrator` 不再负责把 run 从 `queued` 改成 `running`，避免和 Worker 重复写 `run.started`。
