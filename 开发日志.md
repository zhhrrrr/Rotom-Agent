### v1 用户系统

- 新增认证相关文件：
  - `backend/app/api/auth.py`
  - `backend/app/core/security.py`
  - `backend/app/core/deps.py`
  - `backend/app/services/user_service.py`
  - `backend/app/services/auth_service.py`
  - `backend/app/schemas/auth.py`
- 新增接口：
  - `POST /api/auth/register`
  - `POST /api/auth/login`
  - `GET /api/auth/me`
- 密码使用 PBKDF2-SHA256 哈希保存。
- JWT 使用 HMAC-SHA256 签名，登录和注册成功后返回：
  - `access_token`
  - `token_type=bearer`
- `/api/chat` 和 `/api/runs/{run_id}` 改为必须携带 `Authorization: Bearer <token>`。
- `/api/chat` 不再信任请求体里的 `user_id`，统一从 JWT 解析 `current_user`。
- 注册用户时自动创建默认工作区：
  - 名称：`Default Workspace`
  - 路径：`storage/workspaces/{user_id}/default/`
- `/api/chat` 不传 `workspace_id` 时，自动使用当前用户的默认工作区。
- 工具执行根目录从全局 `WORKSPACE_ROOT` 改为当前 run 绑定的 workspace `root_path`，确保不同用户 workspace 隔离。

### 验证

- `python3 -m py_compile` 通过。
- Docker Compose 重建并启动 backend / worker。
- `POST /api/auth/register` 能注册并返回 bearer token。
- `POST /api/auth/login` 能登录并返回 bearer token。
- `GET /api/auth/me` 能根据 bearer token 返回当前用户和默认 workspace。
- 不带 token 调用 `/api/chat` 会被拒绝。
- 带 token 调用 `/api/chat` 能创建 run，worker 能完成任务。
- 验证新用户默认工作区目录已创建，且“列出当前工作区目录”只查看该用户默认 workspace。
