# Rotom Agent 开发日志

## 2026-05-19

### v1 用户与工作区数据库基线

- 在 `main` 的 v0 baseline 基础上创建分支：`feature/rotom-v1-user-gateway`。
- 保持原始 `/home/alpha/projects/v0` 不变，后续 improve 只在 `/home/alpha/projects/v1/Rotom Agent` 进行。
- 增加 `APP_NAME=Rotom Agent`，FastAPI 应用标题改为读取配置。
- Docker 容器名统一为：
  - `rotom-postgres`
  - `rotom-rabbitmq`
  - `rotom-backend`
  - `rotom-worker`
- 数据库新增 v1 表：
  - `users`
  - `workspaces`
  - `model_calls`
  - `event_logs`
- 给 v0 旧表补充 v1 字段：
  - `sessions`: `user_id`, `workspace_id`
  - `messages`: `user_id`, `workspace_id`, `meta`
  - `runs`: `user_id`, `workspace_id`, `current_step`, `retry_count`, `started_at`
  - `tool_calls`: `user_id`, `workspace_id`, `runtime_type`, `risk_level`
- 启动时增加轻量 schema upgrade，兼容已有 v0 数据库。
- 默认创建兼容身份：
  - `user_default`
  - `workspace_default`
- API 保持 v0 兼容：不传 `user_id` / `workspace_id` 时使用默认值。
- 新增事件日志记录：
  - `chat.request.received`
  - `gateway.auth.checked`
  - `workspace.resolved`
  - `session.resolved`
  - `run.created`
  - `run.queued`
  - `run.started`
  - `context.built`
  - `model.call.started`
  - `model.call.completed`
  - `tool.call.started`
  - `tool.call.completed`
  - `run.completed`
  - `run.failed`
- 新增模型调用记录 `model_calls`，保存 request messages、tools、response、token、耗时和状态。

### 验证

- 通过 `python3 -m py_compile` 检查 Python 语法。
- 通过 Docker Compose 重建并启动服务。
- `/health` 返回 `ok`。
- `/api/chat` 能创建 queued run。
- worker 能消费任务，执行工具调用，最终 `/api/runs/<run_id>` 返回 `completed`。
- 验证数据库包含 8 张表：`users`, `workspaces`, `sessions`, `messages`, `runs`, `tool_calls`, `model_calls`, `event_logs`。

### 注意

- 本地 `backend/.env` 包含真实密钥，不进入 Git。
- 运行验收时遇到旧 Docker volume 残留，需要同步 RabbitMQ 密码和创建 `rotom_agent` 数据库；这是本机运行环境问题，不属于源码改动。
- 后续每次代码更新都要继续追加本文件。

### 文档补充

- 新增 `docs/event-log-and-model-call.md`，总结 `event_logs` 和 `model_calls` 的记录时机。
- 明确 `model_calls` 不是只在错误时记录，而是每次真实模型调用都会记录；成功写 `completed`，失败写 `failed`。
- 明确一次工具型 run 通常会产生多条 `model_calls`，例如先让模型选择工具，再让模型根据工具结果生成最终回答。
